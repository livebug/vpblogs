# 内容分类方法——Category 与 Tag 的区别

> 设计博客分类系统时，如何正确使用 Category（分类）和 Tag（标签）？

## 核心区别

| | Category（分类） | Tag（标签） |
|:---|:---|:---|
| **层级关系** | 树状结构，有父子关系 | 扁平化，无层级 |
| **数量约束** | 一篇文章通常 1 个分类 | 一篇文章可以多个标签 |
| **排他性** | 高（一件商品不能同时是"上衣"和"裤子"） | 低（一篇文章可以同时打 `Python` 和 `Docker`）|
| **用途** | 内容归大类 | 跨维度检索 |
| **类比** | 图书馆书架 | 书的索引关键词 |

## 生活类比

买衣服的类比：

- **Category**：衬衫、牛仔裤、帽子——归类到**唯一**的品类
- **Type**：长袖、半袖、花格、白色——同一品类下的**细分属性**

换到博客系统：

- **Category**：`learn`（学习）、`note`（记录）、`think`（心得）、`write`（随笔）——每篇文章归属于一个目录
- **Tag**：`Python`、`Docker`、`VSCode`——一篇文章可以跨分类打多个标签

## 本博客的选择

VitePress 博客采用"目录即分类"的设计：

```
docs/
├── learn/     ← Category: 学习笔记
├── note/      ← Category: 记录（标签页、时间轴等）
├── think/     ← Category: 心得思考
└── write/     ← Category: 随笔写作
```

同时每篇文章通过 frontmatter 的 `tags` 字段实现跨分类的标签检索。

## 何时用 Tag 而不是新建 Category？

- ✅ 内容涉及多个技术栈 → 用 Tag（如 `Docker` + `Python`）
- ✅ 需要跨目录筛选 → 用 Tag
- ❌ 一个主题下只有 1-2 篇文章 → 不需要新建 Category
- ❌ 分类重叠 → 选最贴合的一个 Category，用 Tag 补充
